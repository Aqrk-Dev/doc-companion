## 2. 详细设计

### 2.1 NotebookEdit 覆盖(Fix 1)

**问题**:`hooks/hooks.json` 的 PreToolUse/PostToolUse matcher 仅 `Write|Edit|MultiEdit`,且 gate/remind 只读 `tool_input.file_path`;NotebookEdit 载荷字段是 `tool_input.notebook_path`。后果:把 `.ipynb` 登记为 critical 时,编辑它完全绕过契约门与提醒档(实测 gate exit 0 静默放行)。

**修法**:
- `hooks/hooks.json`:两处 matcher `Write|Edit|MultiEdit` → `Write|Edit|MultiEdit|NotebookEdit`(Stop 无 matcher,不动)。
- `hooks/doc-map-gate.mjs:29`、`hooks/doc-map-remind.mjs:19`:`payload?.tool_input?.file_path` → `payload?.tool_input?.file_path ?? payload?.tool_input?.notebook_path`。
- 其余逻辑(repoRelative/critical 判定/stub)不变——notebook 路径与普通文件路径同构。

### 2.2 子目录会话根发现(Fix 2)

**问题**:三个 hook 用 `payload.cwd`、preflight 用 `--cwd`/`process.cwd()` 直接当仓库根查 `.docc/map.json`。会话在仓库**子目录**启动时,该目录无 `.docc/` → loadDocMap 返回 null → 整套防护静默失效(gate exit 0、preflight 报"缺少 map.json"fail-open),且零告警。

**修法(nearest-ancestor 语义,同 git/.gitignore/package.json 的配置发现惯例)**:
- `hooks/_shared.mjs` 新增 `findDoccRoot(startDir)`:自 `startDir` 起逐级向上,首个存在 `<dir>/.docc/map.json` 的目录即为根并返回;走到文件系统根仍无则返回 `startDir` 原值(保"未 init 零打扰":无 `.docc/` 时行为不变)。纯 `fs.existsSync` + `path.dirname` 循环,任何异常返回 `startDir`,不抛。
- 三个 hook:`const cwd = findDoccRoot(payload?.cwd || process.cwd());`。
- `scripts/doc-preflight.mjs`:解析出 `cwd` 后再 `cwd = findDoccRoot(cwd)`(`--cwd .` 从子目录调用也能向上定位)。
- 下游全部沿用 `cwd` 作根:`repoRelative(filePath, root)` 天然正确(根下文件归一化正确、根外文件仍被拒——安全语义不变);statePath/declarationsPath/hashesPath/ledgerDir/git 子进程 cwd 均以发现根为准 → 会话状态按仓库聚合。
- **边界语义**:只向上找祖先 `.docc/`,不向下找后代。会话若在 monorepo 顶层而 `.docc/` 在某子包内,仍走"未 init 零打扰"——须在该子包(或其子目录)内启动会话。此为有意的最近祖先语义,文档写明。

### 2.3 残留诚实文档化(Fix 3)

**README `## 已知限制`**:
- #2 尾部或新条目:补"同会话并行子 agent 的状态读-改-写在原子写下不会损坏,但竞态窗口内可能重复拦截/重复声明 stub(fail-closed,重试即放行)"——若 #2 已含"多拦一次"则仅补"重复 stub"。
- 新增"CRLF 跨机混配":哈希已 `--no-filters` 消除单机 autocrlf 影响,但共享基线在 autocrlf=true(Windows)与 LF 机器间仍可能对 CRLF 工作树字节报幻影漂移;受影响仓库以 `.gitattributes`(如 `* text=auto eol=lf`)统一工作树行尾。
- 新增"子目录/monorepo":hooks 与 preflight 以**最近祖先** `.docc/map.json` 定位仓库根(v0.4.1 起支持子目录启动);会话须在含 `.docc/` 的目录**或其子目录**内启动,顶层无 `.docc/` 的 monorepo 需在对应子包内启动。
- 新增"平台":原生 Windows `cmd` 执行器下 hooks.json 的 POSIX `sh` 守卫无效,三 hook 失效——用 WSL/git-bash(hooks 非阻塞,失效不影响手动 preflight/postflight)。NotebookEdit 自 v0.4.1 起纳入契约门/提醒档覆盖。

**`.claude-plugin/plugin.json`**:`0.4.0` → `0.4.1`。

**gap-v2 台账** `docs/superpowers/specs/2026-07-11-docc-gap-analysis-v2.md` §3:把 "NotebookEdit 绕过" 与 "子目录静默失效" 标注为 v0.4.1 已处置;Windows 原生 sh、跨会话遗忘声明保留"仍留档";补一行说明"CRLF 跨机混配、同会话状态竞态"降为文档化限制。

## 5. 红线复核

- **fail-open**:findDoccRoot 任何异常返回 startDir;无 `.docc/` 时行为与今日一致(零打扰)。
- **零 npm 依赖 / 引擎零耦合**:仅 node 内置 fs/path;findDoccRoot 无项目特定信息。
- **安全语义不变**:repoRelative 仍以发现根为界拒绝根外文件;子目录发现只是把"根"从 cwd 上移到最近祖先 `.docc/`,不放宽越界判定。
- **同会话同文件只拦一次**:gate 去重逻辑不动。
