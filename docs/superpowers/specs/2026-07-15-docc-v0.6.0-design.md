## 1. 自算归一化哈希(breaking)

`_shared.mjs` 用 `node:crypto`(内置,零 npm 红线无损)替代 git 哈希子进程:

- `hashFileNormalized(cwd, relPosix)` → 读文件 Buffer;若前 8000 字节含 NUL(git 同款二进制启发式)则原字节;否则做 **CRLF→LF 归一化**(仅 `\r\n`→`\n`,lone `\r` 不动);哈希 = `sha1("blob " + len + "\0" + content)`(len 为归一化后字节长)。LF 文件与 `git hash-object --no-filters` **逐字节同值**(已实测)——LF 基线原样兼容,仅 CRLF 文件升级后一次性浮为 modified 候选,verdict 记 formatting 重盖章吸收。读失败返回 null(fail-open,unhashable 告警不变)。
- 批量:`hashFilesNormalized(cwd, rels)` 进程内循环(无子进程、无 maxBuffer 问题)。**删除** `gitHashObject`/`gitHashObjects` 与批量失败回退机制(hashMany 简化为直调)。
- preflight 全部换用;`GIT_MAX_BUFFER` 保留给 ls-files/status,值 64MB→**256MB**。
- SKILL 降级路径注明:手工 `git hash-object` 对 LF 文件等价,CRLF 文件有差异(降级模式接受)。
- 测试:LF 与 git hash-object 交叉一致;CRLF 与其 LF 版本同哈希(幻影漂移消失的端到端用例:同文件 CRLF 化后 preflight 无候选);二进制(含 NUL)不归一化;读失败 null。

## 2. hooks.json exec form(修 Windows)

三个 hook 条目改为:`{"type":"command","command":"node","args":["${CLAUDE_PLUGIN_ROOT}/hooks/doc-map-gate.mjs"],"timeout":15}`(remind/stop-remind 同理)。删除 sh 守卫。README 运行时分层红线措辞更新:hooks 需 node ≥18 于 PATH;无 node 机器每次编辑收到一行非阻塞提示(不再全静默——如实记载);已知限制"平台"条改写:原生 Windows(含无 Git Bash 的 PowerShell 环境)自 0.6.0 起支持。**测试注意**:test/hooks.test.mjs 直接 spawn node 跑 hook 脚本,不经 hooks.json,不受影响;新增一个 hooks.json 形状断言用例(三条目均为 exec form 且 args 引用对应脚本)。

## 3. saveState 合并式写入(修竞态)

`saveState(cwd, sessionId, state)` 写前**重读**磁盘最新状态并合并:`gated`/`reminded` 取并集(保序去重),`notedCount` 取 max,再 atomicWrite。lost-update 窗口从"整个 hook 生命周期"缩到重读与 rename 之间的微秒级;方向仍 fail-closed。返回布尔与告警语义不变。测试:预写状态含 `gated:["src/b.ts"]`,以仅含 `src/a.ts` 的内存 state 调 saveState → 磁盘结果两者兼有;notedCount 取大者。README 已知限制 #2 的"可能多拦一次"措辞弱化为"极小竞态窗口"。

## 4. 共享 gated GC + 小修集

- **写入时同 rel 换键即清旧**:gate 写共享文件时删除同一 `rel@` 前缀的其他键(一文件恒一键)。
- **盖章时 GC**:preflight `--stamp` 成功后,读共享 gated,删除"键中哈希 ≠ 新基线该 rel 哈希"的条目(含 rel 已不在基线的),atomicWrite 回写;失败静默(fail-open,文件本地易失)。
- history 统计命令(README)加撕裂行容错:`.map(l=>{try{return JSON.parse(l)}catch{return null}}).filter(Boolean)`。
- monorepo 顶层 error 文案:preflight 缺 map 的 error 补"或在含 .docc/ 的子包内启动会话"。
- 装饰残留:stampBlocked.message 分隔符统一半角"; ";test/preflight.test.mjs 中过期测试名"--verdict '{}' 表示无漂移处置,不告警(init 场景)"更正为与回退后行为一致的名字。

## 5. `--check` 退出码模式(CI 轨,opt-in)

- `doc-preflight.mjs` 新增 `--check` 标志:报告照常输出;结束时若 `mapIssues.length || driftCandidates.length || docDrift.length` 则 **exit 1**,否则 exit 0。**默认路径(无 --check)恒 exit 0 的 fail-open 红线不变**;`--check` 与 `--stamp` 互斥(同时给报 error exit 2 用法错误——参数错不属 fail-open 范畴)。
- README 新增"CI / pre-commit(可选)"一节:一行接线示例(`node <plugin>/scripts/doc-preflight.mjs --cwd . --check` 于 pre-commit/CI;引擎只发模板不代装)。已知限制"会话外提交"条注明可用 --check 收窄。
- 测试:干净仓库 --check exit 0;有候选 exit 1;--check --stamp 并用 exit 2;无 --check 有候选仍 exit 0(红线回归钉)。
