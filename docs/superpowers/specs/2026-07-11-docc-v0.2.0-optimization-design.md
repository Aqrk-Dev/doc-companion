## 2. 详细设计

### 2.1 锚点精确边界匹配 + 歧义检测

**边界规则(精确定义)**:设锚点为 `anchor`,文档某行为 `line`。命中当且仅当 `line.startsWith(anchor)` 且 `rest = line.slice(anchor.length)` 满足 `rest === ""` 或 `rest` 以空白字符开头。即 `"### 2.6"` 命中 `"### 2.6"` 与 `"### 2.6 accounts 契约"`,不命中 `"### 2.61 其他"` 与 `"### 2.6:标点紧跟"`。

**`hooks/_shared.mjs`**:
- 新增 `countAnchorMatches(cwd, docFile, anchor)` → 返回命中行数(读文件失败返回 0,不抛)。
- `anchorExists` 改为 `countAnchorMatches(...) > 0` 的薄封装,导出 API 与签名不变。

**`scripts/doc-preflight.mjs`**:
- 锚点检查改用 `countAnchorMatches`:0 次 → `锚点未找到`(现状保留);≥2 次 → `mapIssues` 新增一类 `锚点歧义(N 处命中): <file> <anchor>`。
- 用 `Set`(键 `file anchor` 与 `file`)对"文档不存在/锚点未找到/锚点歧义"检查去重——多条目引用同一文档/锚点时不再重复报告(现状会重复,属顺带修正,写入变更说明)。

**兼容性**:既有项目里凑巧被更长标题"字面顶替"的锚点,升级后首跑 preflight 会开始报 `锚点未找到`——这正是本项要暴露的假阳性,属预期行为,须在 README 变更说明中写明。

### 2.2 契约门声明→台账数据通路

**草稿文件**:`.claude/.cache/doc-companion-<sid>.declarations.md`(与会话状态同目录、同 sid 清洗规则)。

**stub 格式**(gate 机器追加,每次拦截一段):

```markdown
## <仓库相对路径> — <ISO8601 时间戳>
- 关联契约锚点: <file anchor>、<file anchor>
- 声明: <!-- pending -->
```

**`hooks/_shared.mjs`**:
- 新增 `declarationsPath(cwd, sessionId)`。
- 新增 `appendDeclarationStub(cwd, sessionId, rel, anchors)` → 建目录、追加 stub、返回布尔;任何异常返回 false,不抛。

**`hooks/doc-map-gate.mjs`**:
- 在 `saveState` 成功之后、`exit 2` 之前调用 `appendDeclarationStub`;写失败仅 `warn`,拦截照常(主流程不依赖 stub)。
- stderr 文案更新:要求 agent 在重试前编辑草稿文件,把 `<!-- pending -->` 替换为声明内容(有影响 → 将同步哪一节;无 → 理由),并给出草稿文件路径。agent 不写也可重试放行(fail-open),但 stub 留下"触发未声明"的硬痕迹。
- 与 §2.5 联动:`.claude/` 在默认排除内,agent 编辑草稿文件不会再次触发 gate/remind,无递归。

**`skills/postflight/SKILL.md`**:
- 台账步骤前插入"合并契约门声明草稿":列出 `.claude/.cache/doc-companion-*.declarations.md`(跨会话草稿一并收);有内容时台账追加条件性第六节:

```markdown
## 契约门声明对账
- <文件>(<时间戳>): <声明内容 或 "触发但未留声明">
```

- 台账写完并确认后删除已合并的草稿文件。
- 纪律节补充:"'触发但未留声明'条目必须当场补写声明或说明原因,不得原样入账了事"。

### 2.3 verified-by 机器标记软校验

**台账模板**(`skills/postflight/SKILL.md`)"受影响调用方/依赖方"节尾追加一行标记,三选一:

```markdown
<!-- verified-by: codegraph_impact -->
<!-- verified-by: grep-fallback -->
<!-- verified-by: unverified -->
```

**`scripts/doc-preflight.mjs --stamp` 时的机检**:
- `git status --porcelain -uall -- <ledgerDir>` (-uall:未提交的新台账在未跟踪目录下也要逐文件列出)(ledgerDir 读 `config.ledgerDir`,默认 `docs/LEDGER`),取状态为新增/修改(非删除)的 `.md` 文件,排除 basename 为 `INDEX.md` 的文件。
- 文件内容不匹配正则 `<!--\s*verified-by:\s*\S[^>]*-->` → 报告 `warnings` 追加 `台账缺 verified-by 标记: <file>`。值不限枚举(枚举仅是模板建议),软校验只查标记存在。
- **`warnings` 为报告新增字段**:字符串数组,始终存在(默认 `[]`),**不影响 `ok`**,不阻塞盖章。

### 2.4 漂移历史 JSONL 埋点

- 常量 `HISTORY_FILE = "docs/DOC-MAP.history.jsonl"` 加入 `hooks/_shared.mjs`(与 DOC_MAP_FILE/HASHES_FILE 并列)。
- `--stamp` 分支中、盖章尝试之后:除非 `config.history === false`(缺省视为 true),追加一行:

```json
{"ts":"<ISO8601>","driftCandidates":0,"docDrift":0,"mapIssues":0,"warnings":0,"stamped":true}
```

- 四个计数(driftCandidates/docDrift/mapIssues/warnings)取本次报告实际值;`stamped` 记录盖章实际结果。写失败 → `warnings` 追加 `历史埋点写入失败: <msg>`,不影响其余流程。
- `history.jsonl` **建议提交入库**(观测数据要跨机器累积才能支撑 P0 触发条件判断),init/README 写明。

### 2.5 默认排除 + `config.exclude`

**`hooks/_shared.mjs`**:
- 新增 `DEFAULT_EXCLUDES = [".understand-anything/", ".claude/"]`,恒生效,不可关闭(无正当场景需要纳管这两个目录)。
- 新增 `getExcludes(docMap)`:返回 DEFAULT_EXCLUDES ∪ `config.exclude` 合法成员。归一化:成员须为非空字符串,trim、去掉开头 `./`、不以 `/` 结尾则补;非数组或非法成员静默忽略(fail-open)。
- 新增 `isExcluded(relPosix, excludes)`:`excludes.some(p => rel.startsWith(p))`。
- `matchEntries(docMap, relPosix)` 入口先判 `isExcluded` → 返回 `[]`。签名不变,gate/remind 两个调用方自动生效。

**`scripts/doc-preflight.mjs`**:
- `listRepoFiles` 结果按 `isExcluded` 过滤后再做 pattern 匹配。
- **显式登记的 `docs[].file` 不受排除影响**(显式登记优先于目录排除)。

**语义边界**:`exclude` 是目录前缀,不是 glob——`**/*_test.go` 这类模式排除仍不支持,须在文档中与 2.7 限制一并写明。

**自引用防护(实现中补充)**:`HASHES_FILE` 与 `HISTORY_FILE` 在 preflight 源侧枚举中恒排除——盖章会重写这两个文件,若被 pattern 覆盖会成为永久漂移候选。hooks 的 `matchEntries` 不排除它们:手工编辑 hashes(若已登记)仍应触发契约门。最终审查补充:docs[].file 侧同样防护——自引用文件登记为文档锚点时报 mapIssue;pattern 仅命中恒排除文件时降为 `warnings` 软告警(升级兼容:v0.1.0 自守护条目不再永久 ok=false);`listDirtyLedgerFiles` 加 `-c core.quotepath=false` 防非 ASCII 台账文件名被引号包裹绕过校验。

### 2.6 文档补全

**README.md**:
- "四环节"表事中行为更新(声明落盘 + 草稿文件)。
- "DOC-MAP 结构"示例与说明补 `config.exclude`、`config.history`;锚点规则更新为精确边界匹配定义。
- 新增"已知限制"节,三条:
  1. glob 不支持 `!` 排除、`{a,b}` 展开、`[abc]` 字符类;目录级排除用 `config.exclude`,模式级排除需拆分正向条目。
  2. 多 agent/worktree 并发编辑同一契约文件时,各会话独立触发契约门、声明互不合并,postflight 合并草稿时需人工对账矛盾声明。
  3. 三档 hook 仅在 Claude Code 会话内生效;会话外提交(人类直接 commit、未装插件的协作者)无门禁,靠下一次 preflight 被动发现(CI 门禁未内置)。
- "报告字段"处补 `warnings` 与 `docs/DOC-MAP.history.jsonl` 说明。

**`skills/init/SKILL.md`**:DOC-MAP 生成模板补 `exclude`/`history` 字段说明;注明锚点边界规则与 glob 限制;history.jsonl 建议入库。

**`skills/preflight/SKILL.md`**:报告字段说明补 `warnings`;锚点歧义作为 `mapIssues` 一类在第 1 步说明中提及。

## 5. 红线复核

- **fail-open 不破坏**:stub 写失败仅 warn、history 写失败进 warnings、verified-by 仅软告警、`getExcludes` 对坏 config 静默忽略——所有新增路径异常时主流程照常。
- **同会话同文件只拦一次**:gated 布尔去重逻辑不变,stub 依附于首次拦截。
- **零 npm 依赖 / 引擎零项目耦合**:全部用 node 内置模块;新增常量与格式均项目无关。
- **契约冻结靠人为裁决**:不引入任何自动改文档内容的行为。
