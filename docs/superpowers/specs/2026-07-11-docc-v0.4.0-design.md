# doc-companion v0.4.0 设计(结构性问题清扫)

- **日期**:2026-07-11
- **依据**:gap-analysis-v2 §3 存档的结构性候选 + v0.3.0 最终审查遗留;用户指令"立刻解决结构性问题"并批准全部关键取舍。1.0 前破坏性变更自由。
- **版本**:0.3.1 → 0.4.0。升级说明:报告 schema 与 glob 语义变化,消费方(自定义脚本)需适配;正常 skill 流程无需操作。

## 1. 范围(五组)

### 2.1 报告 schema 类型化(破坏性)

- `mapIssues`、`warnings` 由 `string[]` 改为 `{ code: string, message: string }[]`。
- **码表(mapIssues)**:`bad-entry`(条目缺 pattern/docs)、`bad-pattern`(非法 pattern)、`pattern-no-match`、`doc-escape`(越出仓库)、`docc-dir-doc`(.docc 内登记)、`doc-missing`、`anchor-missing`、`anchor-ambiguous`、`baseline-corrupt`、`stamp-write-failed`(盖章写入失败,原有行为的类型化)。
- **码表(warnings)**:`pattern-only-excluded`、`unhashable`、`ledger-verified-by-missing`、`verdict-missing`、`history-write-failed`,新增 `verdict-count-mismatch`(§2.3)、`ledger-not-indexed`(§2.5)。
- **盖章拒绝迁出 mapIssues**:新顶层字段 `stampBlocked: { realDrift: number, docDrift: number, mapDefects: number, message: string } | null`(非 stamp 运行恒为 null)。拒绝不再翻转 `ok`、不再计入 history 的 mapIssues 计数。message 保留"盖章已拒绝……--force"文案。
- `ok` 语义不变(mapIssues 为空);history 行计数字段含义不变(计数各自数组长度)。

### 2.2 `--stamp` 受 mapIssues 约束

盖章门扩展:`blockedByMap = mapIssues 中存在 code !== "baseline-corrupt" 的条目`(**豁免 baseline-corrupt**——重新盖章正是基线损坏的修复手段,拦它构成死锁)。`blocked = !force && (真漂移 || docDrift || blockedByMap)`,拒绝时 `stampBlocked.mapDefects` 记非豁免缺陷数。`--force` 跳过全部。

**最终审查修订(C1,用户裁决)**:门改证词感知——拒绝条件 = !force && (非豁免映射缺陷>0 || (存在真实漂移 && 未附相符 verdict))。原"真漂移即拒绝"与 §2.3"候选盖章前不消失"自相矛盾,导致正常周期必须 --force、观测数据被污染;相符 verdict 即对拍完成凭证,--force 回归真正例外。

### 2.3 preflight 归零判据

- SKILL 澄清:`driftCandidates` 相对基线,盖章前不会消失;归零判据 = "mapIssues 为空(baseline-corrupt 类按其指引处理)+ 四类判定计数总和 = 候选数(自我核对)"。
- 机器交叉核验:`--stamp` 且 verdict 非 null 时,`sum(verdict 四键) !== driftCandidates.length` → warning `verdict-count-mismatch`(软)。verdict 为 `{}` 且候选数为 0 视为相符;init 场景(首次纳管全量、verdict `{}`)会触发该告警——SKILL 写明 init 忽略此条或传 `{"other":N}`。**取舍**:init 用 `--verdict '{}'` 时若候选非零将出现一次软告警,接受(提示性质,不阻塞)。

### 2.4 声明草稿生命周期(四条一并)

- **勾稽 ID**(引擎):`appendDeclarationStub` 的 stub 首行改为 `## <rel> — <ISO8601> [id: <sid清洗>#<n>]`,`n` = 本会话已 gated 数(gate 传入)。台账对账节引用 id。
- **两阶段删除**(SKILL):postflight 合并草稿后把草稿**改名**为 `<原名>.merged`(不删);活跃会话后续 stub 落入重建的原名文件,不丢。下次 postflight 开场清理所有 `*.merged`。
- **时序**(SKILL):postflight 中涉及 map.json 编辑的校正动作提前到草稿合并之前;写台账前重新 glob 一次 `doc-companion-*.declarations.md`,有新增并入。
- **跨会话重复**(SKILL):对账节按文件分组,同文件多条声明(不同 id)合并列出。

### 2.5 杂项七项

| 项 | 设计 |
|---|---|
| `a**b` 语义 | `**` 仅在段边界(整段 `**`,即位于串首/`/` 后,且其后是 `/` 或串尾)才跨段;否则视为单星 `[^/]*`(对齐 gitignore)。破坏性,README 说明 |
| 批量哈希 | 新 `gitHashObjects(cwd, rels[]) → { [rel]: sha }`,单次 `git hash-object --no-filters --stdin-paths`(stdin 换行分隔,maxBuffer 同 64MB);**整批失败(status≠0/输出行数不符)→ 回退现有逐文件路径**,保 fail-open 与 `unhashable` 告警粒度。preflight 源侧与 doc 侧均改用。`gitHashObject` 保留(回退路径与降级文档用) |
| history 轮转 | `--stamp` 追加后若行数 > 上限(`config.historyLimit`,默认 500,`0` = 不限)→ 保留最新上限行数,tmp+rename 原子重写;失败进 warnings(`history-write-failed` 复用) |
| version 激活 | `loadDocMap`:`version` 缺省或 `=== 1` 通过,否则返回 null(hooks 静默零打扰,fail-open);preflight 对 loadDocMap 失败时预读原始 JSON,若因 version 不符则 error 文案为 `map.json version <X> 不受支持(当前引擎支持 1)` |
| INDEX.md 机检 | `--stamp` 时对每个本轮新增/修改台账(复用 listDirtyLedgerFiles),检查 `<ledgerDir>/INDEX.md` 内容包含其文件名 → 缺失报 warning `ledger-not-indexed`(INDEX.md 不存在时同样告警) |
| rename 行解析 | listDirtyLedgerFiles:状态含 `R`/`C` 的行取 ` -> ` 之后路径 |
| 降级路径等价性 | 0.3.0 已修;本轮仅核验三个 SKILL 降级段与引擎语义一致,不一致处随 §2.3/2.4 的 SKILL 改动一并修正 |

## 3. 配套

- README:报告字段(typed schema + stampBlocked + 新告警码)、glob 语义变更、historyLimit、升级说明(0.3.x→0.4.0:schema/glob 破坏性,消费方适配;skill 流程无操作)。
- 三个 SKILL:preflight(归零判据重写、typed 字段阅读方式)、postflight(草稿生命周期重排、对账 id/分组、INDEX 告警处置)、init(historyLimit 说明、verdict-count-mismatch 说明)。
- 测试:全量迁移 typed 断言(以 code 为主、message 为辅);新增——stampBlocked 结构、map 缺陷挡盖章与 baseline-corrupt 豁免、verdict-count-mismatch、stub id 格式、`a**b` 单星、批量哈希等价+回退(悬空 symlink 混入)、轮转(超限裁剪/0 不限)、version 不符(hooks 静默+preflight 精确 error)、INDEX 告警、rename 行。
- 版本 0.4.0;gap-v2 §3 对应条目标注落地。

## 4. 红线复核

- fail-open:批量哈希失败回退逐文件;version 不符 hooks 静默;轮转失败软告警;stampBlocked 仍 exit 0。
- 同会话同文件只拦一次:stub id 使用 gated 计数,不改变去重语义。
- 零 npm 依赖 / 零项目耦合:不变。
- `--force` 保留人的最终决定权;码表是机器可读层,message 仍面向人。
