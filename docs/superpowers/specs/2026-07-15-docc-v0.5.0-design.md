## 1. 详细设计

### 1.1 driftCandidates 结构化(P0,breaking)

**现状**:`driftCandidates: string[]` 裸路径——agent 必须手工重推 glob 才知道候选牵着哪些锚点,也分不清"修改/首次纳管/已删",是 verdict 分类的最大错误面。

**新 schema**(元素,按 file 排序):

```json
{ "file": "src/api/a.ts", "kind": "modified", "docs": [ { "file": "docs/a.md", "anchor": "## 契约", "critical": true, "note": "API 契约" } ] }
```

- `kind`:`"modified"`(基线有记录且失配)| `"new"`(无基线记录,首次纳管)| `"removed"`(基线有记录但本次未纳管——反向对账产出)。
- `docs`:该文件命中的**所有** entry 的 docs 合并去重(键 `file+anchor`),字段 `file/anchor/critical/note`(anchor/note/critical 缺省则省略该键;critical 仅在 true 时输出)。`removed` 的 docs 用其记录路径对全部 pattern 匹配求得(pattern 匹配字符串,不需文件存在)。
- `realSourceDrift` 计数 = kind ∈ {modified, removed} 的个数(语义不变);`--verdict` 相符判据仍为计数总和 == driftCandidates.length(不变)。
- **连带更名**(同一 breaking 窗口):`stampBlocked` 内 `realDrift`→`realDriftCount`、`docDrift`→`docDriftCount`(消除与顶层 `docDrift` 数组的同名异型歧义)。`docDrift` 顶层保持 string[](文档侧无需结构化)。
- **同步**:README 报告字段节;`skills/preflight/SKILL.md` 第 2 步改为"直接读候选的 `docs` 与 `kind`,不再对照 map 手工求锚点"(删手工推导步);`skills/postflight/SKILL.md` 第 2 步同理改为报告驱动。

### 1.2 verdict 防呆(P0)

现状:`--verdict "{"docLag":1}"`(双引号误用)JSON.parse 失败被静默当"未提供"→ 报"证词缺失",误导排障;未知键静默丢弃。

- 解析处记录两个标志:parse 抛错 → stamp 块内 `warnings` 增 `{code:"verdict-parse-failed", message:"--verdict 参数解析失败(非法 JSON,已按未提供处理;注意 shell 引号转义,建议单引号包裹)"}`;raw 对象中存在四个已知键之外的键 → `{code:"verdict-unknown-key", message:"--verdict 含未知键(已忽略): k1, k2(合法键:formatting/docLag/codeViolation/other)"}`。
- 两码加入 README 码表。仅 `--stamp` 时产出(与 verdict-missing 一致)。

### 1.3 代码修缮四件套(P1)

1. **`atomicWrite(filePath, content)`** 提炼进 `_shared.mjs`(tmp=`${p}.${pid}.tmp`、mkdir、write、rename、finally 清理;失败原样抛,调用方自行 catch 定文案)。三处替换:`saveState`、盖章写 hashes、history 轮转——**顺带修复 history 轮转缺 finally 清理**(现三处中唯一无清理)。
2. **锚点格式化统一 + 修 gate 丢 note**:`_shared.mjs` 增 `formatDocRef(d)` → `` `${d.file} ${d.anchor ?? ""}${d.note ? `（${d.note}）` : ""}`.trim() ``;gate(现丢 note)与 remind 都改用——契约门更需要上下文,note 补上是行为修正。
3. **pushOnce 工厂**:preflight 的 `pushIssueOnce`/`pushWarningOnce` 收敛为 `makePushOnce(arr)` 工厂。
4. **gate saveState 失败降级放行补集成测试**:hooks 级用例——`.claude/.cache` 预置为普通文件使 saveState 失败,gate 对 critical 文件应 exit 0(fail-open 红线唯一缺集成测试的分支)。

### 1.4 跨会话 gated 去重(P1)

**现状**:gated 按 session_id 分文件,换会话必重拦 + 重写 stub,多会话大改动每换一次会话交一次税(README #2 部分记载)。

**设计**:去重键升级为 **`<rel>@<基线哈希>`**,存跨会话共享文件。

- 共享文件:`.claude/.cache/doc-companion-gated.json`(发现根下;`.claude/.cache/` 已 gitignore),形如 `{"src/a.ts@abc123":true}`。
- 键:`` `${rel}@${recSources[rel] ?? "unbaselined"}` ``,`recSources` 读 `.docc/hashes.json` 的 `sources`(readJsonSafe,fail-open:读不到=空对象→键用 `unbaselined`)。
- **盖章即自然失效**:stamp 重写 hashes → 该文件基线哈希变 → 旧键不再命中 → 下轮改动重新设防。零清理逻辑。
- gate 流程:会话内 `state.gated` 快路径保留(不读盘先查);未命中再查共享文件键;两者都无 → 拦截,同时写 session state(现状)**和**共享文件(atomicWrite;写失败仅 warn,会话内去重仍兜底)。命中共享键 → exit 0,**不写 stub**(声明已在首拦会话留过,正确行为)。
- `_shared.mjs` 新增:`sharedGatedPath(cwd)`、`loadSharedGated(cwd)`(损坏/缺失→`{}`)、`gatedKey(rel, recSources)`;gate 接线。
- 并发:最后写者赢(与现状一致,README #2 语义不变)。
- **测试**:sid1 拦 src/a.ts → sid2 同文件 exit 0 且 declarations-sid2 无 stub;模拟盖章(改 hashes.json 中该文件哈希)→ sid3 重新拦;无 hashes.json(从未盖章)→ 跨会话仍去重(unbaselined 键);共享文件损坏 → fail-open 重拦(安全方向)。
