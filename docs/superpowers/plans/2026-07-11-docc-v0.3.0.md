# doc-companion v0.3.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 按 `docs/superpowers/specs/2026-07-11-docc-v0.3.0-design.md` 实现四项破坏性变更:`.docc/` 独立数据目录(含内置自守护与整类特判删除)、exclude 模式级排除、盖章一致性(范围改源+盖章门)、history 携带 `--verdict` 语义判定。

**Architecture:** 改动集中在既有骨架:`hooks/_shared.mjs`(常量/排除/glob)、`hooks/doc-map-gate.mjs`(内置自守护)、`scripts/doc-preflight.mjs`(特判删除/盖章门/--verdict)、三个 SKILL.md 与 README 重写、全部测试 fixture 迁移到 `.docc/` 布局。无兼容层(用户裁决:1.0 前破坏性变更自由,升级=重新 init)。

**Tech Stack:** Node ≥18 内置模块,`node --test` + tmpdir,零 npm 依赖。

## Global Constraints

- **零 npm 依赖**:只用 `node:` 内置模块。
- **fail-open 红线**:盖章门拒绝仍 exit 0 且只影响盖章动作;`--verdict` 缺失仅软告警;坏 glob 成员静默忽略;内置守护复用既有 gate 流程(状态不可写降级放行不变)。
- **同会话同文件只拦一次**:`state.gated` 布尔去重语义不变。
- **无兼容层**:不读旧位置 `docs/DOC-MAP.json`,不留 0.2.1 旧键兼容垫;升级说明写"重新 init"。
- **测试命令**:仓库根目录 `node --test`(不带路径参数)。
- **中文注释/文案**;版本 bump(0.2.1 → 0.3.0)只在 Task 7 做;marketplace.json 无版本字段不改。

(b) `getExcludes` / `isExcluded` 整体替换为:

```js
/**
 * 默认排除 ∪ config.exclude。成员含 * 或 ? 走极简 glob 整路径匹配,
 * 否则按目录前缀(归一化:trim、去 ./、补尾 /);坏输入/坏 glob 静默忽略(fail-open)。
 */
export function getExcludes(docMap) {
  const prefixes = [...DEFAULT_EXCLUDES];
  const regexps = [];
  const extra = Array.isArray(docMap?.config?.exclude) ? docMap.config.exclude : [];
  for (const p of extra) {
    if (typeof p !== "string" || !p.trim()) continue;
    let s = p.trim();
    if (s.startsWith("./")) s = s.slice(2);
    if (!s) continue;
    if (/[*?]/.test(s)) {
      try {
        regexps.push(globToRegExp(s));
      } catch {
        // 坏 glob 静默忽略
      }
    } else {
      prefixes.push(s.endsWith("/") ? s : `${s}/`);
    }
  }
  return { prefixes, regexps };
}

export function isExcluded(relPosix, excludes) {
  return (
    excludes.prefixes.some((p) => relPosix.startsWith(p)) ||
    excludes.regexps.some((rx) => rx.test(relPosix))
  );
}
```

(matchEntries 与 preflight 的调用点把 `getExcludes` 返回值原样传给 `isExcluded`,无需改动。)

| 事前 | `/docc:preflight` | 哈希比对锁定漂移候选(修改/首次纳管/**已删除**/文档被直改),只对候选做语义对拍并按四类记数(formatting/docLag/codeViolation/other),漂移归零再开工 |
| 事中 | hooks 自动 | 改到映射文件→提醒一次(每文件每会话);改 `critical` 文件或 `.docc` 数据文件→拦一次要求声明文档影响,同时机写声明 stub 到 `.claude/.cache/doc-companion-<session>.declarations.md` 供 postflight 对账,重试放行;每轮收尾→未处置清单一行提醒(不阻塞) |
| 事后 | `/docc:postflight` | 以机检报告为核对清单(与盖章范围同口径)→ 合并声明草稿 → 五字段台账(+声明对账节,verified-by 标记)→ 索引校正 → 盖章(带 `--verdict` 判定) |
| 索引 | preflight/postflight 内建 | 锚点悬空、锚点歧义、pattern 失配、孤儿清理 |

## 盖章门

`--stamp` 时若存在**真漂移**(有基线记录且失配、或已删除)或 `docDrift`,盖章被拒绝(`stamped:false` + mapIssue),防止"没走完对拍就盖章"把漂移静默吸收进基线;首次纳管不受影响。例外登记场景(如经用户裁决暂留的代码违约)用 `--force` 显式跳过,`forced` 记入 history。

## 设计红线

- **fail-open**:任何脚本异常、状态损坏、运行时缺失 = 放行 + 告警,绝不阻塞开发;契约门状态不可写时降级放行;盖章门拒绝仍 exit 0,只影响盖章动作本身。
- **同会话同文件只拦一次**:迭代式"编辑→测试→再编辑"不受契约门干扰。
- **运行时分层**:核心链路只依赖 git;hooks 需要 node(≥18,零 npm 依赖),无 node 静默哑火,skill 有手工降级路径。
- **引擎零项目耦合**:本仓库代码不含任何具体项目信息。
- **契约冻结靠人为裁决**:工具只记录、比对、催促、留证,绝不代写文档;`--force` 保留人的最终决定权。

## map.json 结构

```json
{
  "version": 1,
  "config": {
    "ledgerDir": ".docc/LEDGER",
    "exclude": ["ent/", "**/*_pb.go"],
    "history": true
  },
  "entries": [
    {
      "pattern": "src/api/**",
      "docs": [
        { "file": "docs/appendix.md", "anchor": "### 2.6", "note": "API 契约", "critical": true },
        { "file": "README.md", "anchor": "## 状态" }
      ]
    }
  ]
}
```

- `pattern`:极简 glob(`**` 跨段 / `*` 段内 / `?` 单字符;连续 `**/` 自动折叠)。
- `anchor`:标题行前缀,**边界匹配**(前缀后须行尾或空白);命中 ≥2 处报"锚点歧义"。
- `critical`:事中契约门开关。
- `config.exclude`:排除数组——成员含 `*`/`?` 按 glob 整路径匹配(如 `**/*_pb.go`),否则按目录前缀(如 `ent/`);叠加在默认排除(`.understand-anything/`、`.claude/`、`.docc/`)之上。适用于**入库的**生成物;未入库的(.gitignore 覆盖)本来就不纳管。
- `config.history`:缺省 true——盖章时向 `.docc/history.jsonl` 追加一行 `{ts,driftCandidates,docDrift,mapIssues,warnings,stamped,forced,verdict}`;`verdict` 来自 `--verdict`(四类判定计数),用于回答"格式化误报是否真实出现"等长期问题;建议随仓库提交。

## 报告字段

`scripts/doc-preflight.mjs [--stamp] [--force] [--verdict '<json>'] [--cwd <dir>]` 输出 JSON:`ok`(mapIssues 为空)、`driftCandidates`(修改/首次纳管/已删除)、`docDrift`、`mapIssues`(含锚点歧义、路径越界、.docc 内登记、基线损坏、盖章拒绝)、`warnings`(软告警:仅命中被排除文件、无法哈希、台账缺 verified-by、缺 --verdict;不影响 `ok`)、`stamped`。

## 已知限制

1. **glob 无排除语法**:pattern 不支持 `!`/`{a,b}`/`[abc]`;排除需求走 `config.exclude`。
2. **多 agent/worktree 并发不合并声明**:并行会话各写各的声明草稿,postflight 合并时需人工对账;同会话并行子 agent 仍有状态竞态窗口(原子写保证不损坏,可能多拦一次)。
3. **仅 Claude Code 会话内生效**:会话外提交无门禁,靠下一次 preflight 被动发现(CI 门禁未内置);`.docc/hashes.json` 被会话外手工篡改无实时防护,靠 git 历史审计。
4. **从 0.2.x 升级(破坏性)**:数据文件迁至 `.docc/`、diffBase 删除、history 行新增 forced/verdict 字段——删除旧 `docs/DOC-MAP*` 文件(台账可手工移至 `.docc/LEDGER/`)后重新 `/docc:init`。1.0 前版本均可能破坏性变更。
